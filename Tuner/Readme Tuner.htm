Summary:

 

This simple example assembly illustrates the use of an assembly motion ratio constraint. Unlike other constraints which REQUIRE a topological relationship between parts, the motion constraint does not.

 

The example is of a guitar tuner (a small, manual gearbox). In this example, the actual gears were not modeled however the gear RATIO between the input and output is 16:1 - that is for every sixteen turns of the button, the string post makes a single revolution. This allows a precise amount of tension to be applied to the guitar string.

 

If you select the "+" in the tree structure next to the component named: button.ipt:1 in the assembly browser and expand you will see rotational motion constraint (with it's ratio) and an angle constraint.

 

Select the angle constraint. Right-mouse click to open up the context menu and select DRIVE CONSTRAINT. Inventor "remembers" how a constraint was last driven, and I have already entered values that will drive this angle from 0 to 350 degrees in increments of 10 degrees for a total of three times.

 

Click the "run" arrow (the left-most arrow on the interface) to watch the button turn and the post turn in conjunction with the button. NOTE: you may want to adjust your viewing angle so that you can see the hole in the string post where the string would actually be anchored - so that you can see that the post is also moving.

 

Because the constraint drive only turns the button a full cycle three times, the post only travels approximately 65 degrees (or 3/16ths of 360 degrees). You can change the value I have used by expanding the drive constraint dialog using the ">>" button on the lower right corner of the dialog and changing the "Repetitions" value from 3 to another positive integer.

 

If you suppress the angle constraint (by right-mouse clicking on the constraint in the browser and then selecting SUPPRESS from the context menu), you can simply grab the button with your mouse and turn it, causing the post to turn.

 

The button must make 16 complete cycles for the post to make one.

 

This illustrates the ability to couple assembly components using the motion constraint despite the fact that the actual (positioned and constrained) parts are not even touching.
